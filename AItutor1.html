I'm finding Python decorators a bit confusing. Can you explain how they actually work under the hood?

smart_toy

Think of a decorator as a wrapper for a function. It allows you to add functionality before or after the original function runs, without changing the function's code itself.

lightbulb

Analogy: Gift Wrapping

Imagine a function is a plain toy. A decorator is like wrapping paper. You wrap the toy (pass the function) inside the paper (the decorator), and what you get back is a wrapped gift. It's still the toy inside, but now it has an extra layer!

example.py
def my_decorator(func): def wrapper(): print("Before...") func() print("After...") return wrapper @my_decorator def say_hello(): print("Hello!")
